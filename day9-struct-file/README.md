## ✅ 오늘의 정리

| 개념 | 설명 |
|------|------|
| 구조체(struct) | 여러 변수(멤버)를 묶어 하나의 단위로 저장 |
| 구조체 배열 | 같은 구조체 타입을 연속적으로 저장하는 배열 |
| &struct.member | 멤버의 메모리 주소 확인 |
| sizeof(struct) | 구조체가 차지하는 총 메모리 바이트 수 |
| 패딩(Padding) | 구조체 멤버 간 메모리 정렬로 생기는 빈 공간 |
| Little-endian | 낮은 주소에 LSB(Least Significant Byte) 저장 |
| 파일 입출력 | fwrite/fread로 구조체 배열을 파일에 저장/읽기 |
| GDB | 구조체 내부 메모리와 주소 추적 |
| x/xb | 메모리 내용을 바이트 단위로 출력 |

---

## 💬 요약 정리
> 구조체 메모리는 단순 나열이 아니라 **정렬**에 의해 배치됨  
> 구조체 배열과 파일 입출력을 통해 데이터를 연속적으로 관리 가능  
> double 타입은 **8바이트 + Little-endian 방식**으로 메모리에 저장  
> GDB와 x/xb 명령어로 실제 메모리 구조를 확인하며 패딩과 주소 배치를 이해
